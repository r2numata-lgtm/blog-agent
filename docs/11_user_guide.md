## 📖 はじめに

Blog Agentは、WordPress Gutenberg完全対応のAI記事生成Webアプリケーションです。このガイドでは、Blog Agentの基本的な使い方を説明します。

### Blog Agentでできること

- AI（Claude）による高品質な記事の自動生成
- タイトル候補の自動提案（3案）
- チャットベースでの記事修正
- WordPressにそのまま貼り付けられるGutenbergブロック形式での出力
- 記事の保存・管理・エクスポート

---

## 🚀 クイックスタート

### 1. アカウント登録

1. Blog Agentにアクセス
2. 「新規登録」をクリック
3. メールアドレスとパスワードを入力
4. 確認メールに記載されたコードを入力

### 2. 初期設定

1. ログイン後、「設定」ページにアクセス
2. 記事スタイルを設定（任意）
   - テイスト（フォーマル/カジュアル/親しみやすい/専門的）
   - 一人称（私/僕/筆者など）
   - 読者への呼びかけ（あなた/みなさんなど）
3. サンプル記事をアップロード（任意）
4. 装飾プリセットを設定（任意）

### 3. 記事生成

1. 「新規記事」をクリック
2. 記事の要点を入力
3. 「生成」ボタンをクリック
4. AIが記事を生成します（約30秒〜1分）
5. タイトル候補から選択
6. 必要に応じてチャットで修正

### 4. エクスポート

1. エディタ画面で「エクスポート」をクリック
2. 出力形式を選択
   - Gutenbergブロック（WordPress用）
   - HTML
   - JSON
3. コピーまたはダウンロード

## 📝 記事を生成する

### 入力項目

| 項目 | 必須 | 説明 |
|------|------|------|
| 記事の要点 | ○ | 記事で伝えたい主要なポイント（箇条書き推奨） |
| 対象読者 | - | 記事の想定読者（例：初心者向け、専門家向け） |
| キーワード | - | SEOキーワード（カンマ区切り） |
| 記事タイプ | - | 情報提供/ハウツー/レビュー/比較 |
| 文字数 | - | 目安文字数（1000〜5000文字） |
| 内部リンク | - | 挿入したい既存記事のURL |

### 要点の書き方のコツ

**良い例：**
```
・WordPressテーマ「Cocoon」のインストール方法
・管理画面からのテーマ設定手順
・初心者がつまずきやすいポイント
・スキンの選び方と適用方法
・高速化のための基本設定
```

**悪い例：**
```
Cocoonについて書いて
```

### タイトル選択

AIが3つのタイトル案を提案します：

1. **注目を引くタイトル** - クリック率重視
2. **SEO重視タイトル** - 検索エンジン最適化
3. **わかりやすいタイトル** - 内容が明確

各タイトルには「狙い」の説明が付いているので、目的に合ったものを選んでください。

## 💬 チャットで修正する

### 基本的な使い方

エディタ画面右側のチャットパネルから、自然な言葉で修正指示ができます。

**修正指示の例：**
- 「導入部分をもっと引き込むような文章にして」
- 「3番目の見出しの内容をもう少し詳しく説明して」
- 「全体的にもう少しカジュアルな文体にして」
- 「この部分に具体例を追加して」

### 部分修正

特定の部分だけを修正したい場合：

1. 修正したいテキストを選択
2. チャットで「選択した部分を〜に修正して」と指示

### 修正履歴

- 最大10件の修正履歴が保存されます
- 「元に戻す」ボタンで以前の状態に戻せます
- 差分表示で変更箇所を確認できます

## ✏️ エディタで編集する

### ブロック操作

**ブロックの追加**
1. 「+」ボタンをクリック
2. ブロックタイプを選択
   - 基本：段落、見出し、リスト、引用
   - レイアウト：グループ、カラム、テーブル
   - カスタム：情報ボックス、吹き出し

**ブロックの移動**
- ドラッグ&ドロップで順序変更
- 上下矢印ボタンで移動

**ブロックの削除**
- ブロックを選択して「削除」をクリック

### キーボードショートカット

| ショートカット | 機能 |
|---------------|------|
| Cmd/Ctrl + Z | 元に戻す |
| Cmd/Ctrl + Shift + Z | やり直す |
| Cmd/Ctrl + S | 保存 |
| Cmd/Ctrl + C | コピー |
| Cmd/Ctrl + V | 貼り付け |

### プレビューモード

- **編集モード**: ブロックを編集可能
- **プレビューモード**: 実際の表示を確認

レスポンシブ確認：
- PC表示
- タブレット表示
- モバイル表示

## 📁 記事を管理する

### 記事一覧

「記事一覧」ページでは、作成した記事を管理できます。

**表示される情報：**
- タイトル
- ステータス（下書き/公開済み）
- 文字数
- 作成日/更新日

### 検索・フィルタ

- **検索**: タイトル、本文、キーワードで検索
- **フィルタ**: ステータス（全て/下書き/公開済み）
- **ソート**: 更新日/作成日/タイトル

### 一括操作

1. 記事を選択（チェックボックス）
2. 一括操作を選択（削除など）
3. 実行

## 📤 エクスポートする

### 出力形式

#### 1. Gutenbergブロック形式（推奨）
WordPressにそのまま貼り付けられる形式です。

**使い方：**
1. 「Gutenbergブロック」を選択
2. 「コピー」をクリック
3. WordPress管理画面でCtrl+V（Cmd+V）で貼り付け

#### 2. HTML形式
装飾・スタイル付きのHTMLです。

**用途：**
- WordPress以外のブログ
- メールニュースレター
- 静的サイト

#### 3. JSON形式
記事データ（本文+メタ情報）の構造化データです。

**用途：**
- バックアップ
- 他システムへの連携
- データ分析

### メタ情報

エクスポート時に以下のメタ情報も出力されます：
- タイトル
- メタディスクリプション
- メタキーワード
- 文字数
- 生成日時

## ⚙️ 設定

### 記事スタイル設定

生成される記事の文体を統一できます。

| 設定項目 | 選択肢 |
|---------|--------|
| テイスト | フォーマル/カジュアル/親しみやすい/専門的 |
| 一人称 | 私/僕/筆者/自分 など |
| 読者呼びかけ | あなた/皆さん/読者の方 など |
| トーン | 客観的/主観的/説明的 など |
| 導入スタイル | 問いかけ/結論ファースト/ストーリー など |

### サンプル記事

過去に書いた記事をアップロードすると、AIがその文体を学習します。

**対応形式：**
- HTML（.html）
- Markdown（.md）

**制限：**
- 最大3記事
- 1記事あたり最大100KB

### 装飾プリセット

記事内で使用する装飾ブロックを設定します。

**設定可能な装飾：**
- 情報ボックス（青）
- 警告ボックス（黄）
- 成功ボックス（緑）
- 吹き出し
- 引用ブロック
- 表

### SEO設定

- メタディスクリプションの文字数上限
- キーワード数の上限

## ❓ よくある質問

### 記事生成について

**Q: 記事の生成にどのくらい時間がかかりますか？**
A: 通常30秒〜1分程度です。文字数が多い場合はより長くかかることがあります。

**Q: 生成された記事の著作権は？**
A: 生成された記事の著作権はユーザーに帰属します。自由にご利用いただけます。

**Q: 同じ要点で再生成すると同じ記事になりますか？**
A: いいえ、毎回異なる記事が生成されます。気に入るまで再生成できます。

### 編集について

**Q: WordPressにコピーしたらレイアウトが崩れました**
A: Gutenbergブロック形式でコピーしていることを確認してください。また、WordPressのテーマによっては独自のスタイルが適用される場合があります。

**Q: 保存したのに記事が消えました**
A: ブラウザのローカルストレージに保存されています。同じブラウザでアクセスしてください。シークレットモードでは保存されません。

### アカウントについて

**Q: パスワードを忘れました**
A: ログイン画面の「パスワードをお忘れの方」からリセットできます。

**Q: 退会したい場合は？**
A: 設定画面から退会手続きができます。
