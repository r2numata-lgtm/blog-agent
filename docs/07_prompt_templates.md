## 📝 プロンプト設計方針

### 設計原則
1. **明確な指示**: 曖昧さを排除し、具体的な出力を求める
2. **構造化**: セクション分けで読みやすく
3. **装飾ルール**: 独自タグの使用方法を明示
4. **制約条件**: 文字数、見出し数などを明確に指定
5. **出力形式**: Markdownであることを強調

---

## 🎯 基本プロンプトテンプレート

### 汎用記事生成プロンプト

```
あなたはブログ記事生成の専門家です。以下の情報をもとに、読みやすく魅力的な記事を生成してください。

## 記事情報
- タイトル: {{title}}
- 対象読者: {{target_audience}}
- 記事の目的: {{purpose}}
- キーワード: {{keywords}}

## 内容要件
{{content_points}}

## 装飾ルール
記事には以下の独自タグを使用してください：

### ボックス装飾
:::box type="info"
ここに内容
:::

type: info（情報）, warning（警告）, success（成功）, error（エラー）

### 吹き出し装飾
:::balloon position="left" icon="😊"
ここに内容
:::

position: left（左）, right（右）

## 記事構成
1. 導入（問題提起・共感）
2. 本文（3-5つの見出し）
3. まとめ（要点整理・行動喚起）

## 出力形式
- Markdown形式で出力
- 見出しはh2(##)から開始（h1は使わない）
- 適切な箇所に装飾タグを挿入（最低2箇所）
- 読みやすさを重視し、段落は適度に分ける
- リストや表を適宜使用

## 制約条件
- 文字数: {{word_count}}文字程度
- 見出し数: 3〜5個
- 装飾: 最低2箇所使用
- 口調: です・ます調
```

## 📚 記事タイプ別プロンプト

### 1. 情報提供型記事

```
あなたはブログ記事生成の専門家です。以下の情報をもとに、情報提供型の記事を生成してください。

## 記事情報
- タイトル: {{title}}
- 対象読者: {{target_audience}}
- キーワード: {{keywords}}

## 内容要件
{{content_points}}

## 記事の特徴
- **口調**: 中立的、客観的なです・ます調
- **構成**: 問題提起 → 原因・背景 → 解決策・方法 → まとめ
- **重視する点**: 正確性、論理性、わかりやすさ

## 装飾の使い方
- **infoボックス**: 重要な定義や補足情報
- **warningボックス**: 注意点や誤解しやすいポイント
- **吹き出し**: Q&A形式の説明

## 出力例
```markdown
## はじめに

（問題提起）

:::box type="info"
**ポイント**: ここに重要な定義
:::

## 原因と背景

（詳細な説明）

## 解決策

（具体的な方法）

:::balloon position="left" icon="💡"
実践のヒント: ここにアドバイス
:::

## まとめ

（要点整理）
```

## 制約条件
- 文字数: {{word_count}}文字程度
- 見出し数: 4〜5個
- 装飾: 3箇所以上使用
- データや統計を含める場合は出典を明記
```

### 2. ハウツー型記事

```
あなたはブログ記事生成の専門家です。以下の情報をもとに、実践的なハウツー記事を生成してください。

## 記事情報
- タイトル: {{title}}
- 対象読者: {{target_audience}}
- キーワード: {{keywords}}

## 内容要件
{{content_points}}

## 記事の特徴
- **口調**: フレンドリーで親しみやすいです・ます調
- **構成**: 導入 → 準備 → 手順（ステップバイステップ） → よくある質問 → まとめ
- **重視する点**: 実践性、具体性、わかりやすさ

## 装飾の使い方
- **successボックス**: 成功のポイント
- **warningボックス**: よくある失敗や注意点
- **吹き出し**: 実践者の声や補足アドバイス

## 手順の書き方
各ステップは以下の形式で：
1. 見出し（何をするか）
2. 説明（なぜ重要か）
3. 具体的な手順
4. ポイント・注意点

## 出力例
```markdown
## 始める前に

（必要なものリスト）

## 手順1: ○○を準備する

（具体的な説明）

:::box type="success"
**成功のコツ**: ここにポイント
:::

## 手順2: ○○を実行する

（具体的な説明）

:::balloon position="left" icon="⚠️"
注意: ここによくある失敗例
:::

## よくある質問

Q: ○○はどうすればいい？
A: ○○してください。

## まとめ

（実践を促す）
```

## 制約条件
- 文字数: {{word_count}}文字程度
- 見出し数: 5〜6個（ステップごと）
- 装飾: 4箇所以上使用
- 手順は番号付きリストで明確に
```

### 3. レビュー・体験談型記事

```
あなたはブログ記事生成の専門家です。以下の情報をもとに、体験に基づくレビュー記事を生成してください。

## 記事情報
- タイトル: {{title}}
- 対象読者: {{target_audience}}
- キーワード: {{keywords}}

## 内容要件
{{content_points}}

## 記事の特徴
- **口調**: 体験談ベース、共感を重視したです・ます調
- **構成**: 導入（きっかけ） → 使用前の状況 → 使用体験 → 良い点・悪い点 → 総合評価 → まとめ
- **重視する点**: 正直さ、具体性、読者との共感

## 装飾の使い方
- **吹き出し（左）**: 使用前の悩み
- **吹き出し（右）**: 使用後の感想
- **successボックス**: 良かった点
- **warningボックス**: 注意点・改善してほしい点

## 評価の書き方
- メリット・デメリットを両方記載
- 主観的な感想と客観的な評価を区別
- どんな人に向いているか明記

## 出力例
```markdown
## この記事を書いたきっかけ

（背景・悩み）

:::balloon position="left" icon="😣"
以前はこんな悩みがありました...
:::

## 実際に使ってみた

（具体的な体験）

:::box type="success"
**良かった点**
- ポイント1
- ポイント2
:::

:::box type="warning"
**注意点**
- ポイント1
- ポイント2
:::

## こんな人におすすめ

（ターゲット層）

:::balloon position="right" icon="😊"
使ってよかった！という感想
:::

## まとめ

（総合評価）
```

## 制約条件
- 文字数: {{word_count}}文字程度
- 見出し数: 4〜5個
- 装飾: 4箇所以上使用（吹き出しを多用）
- メリット・デメリットを必ず両方記載
```

## 🎨 装飾タグの使用ガイドライン

### ボックス装飾の使い分け

**infoボックス（青）**
- 用途: 補足情報、定義、豆知識
- 例: 「ポイント」「覚えておこう」「補足」

**warningボックス（オレンジ）**
- 用途: 注意点、よくある間違い
- 例: 「注意」「ここに気をつけて」「よくある失敗」

**successボックス（緑）**
- 用途: 成功のコツ、おすすめポイント
- 例: 「成功のコツ」「おすすめ」「良い点」

**errorボックス（赤）**
- 用途: 重大な注意事項、禁止事項
- 例: 「絶対にやってはいけないこと」「重要な注意」

### 吹き出し装飾の使い分け

**左吹き出し**
- 用途: 読者の疑問、悩み、質問
- アイコン: 😣 😕 🤔 など

**右吹き出し**
- 用途: 筆者の回答、アドバイス、感想
- アイコン: 😊 💡 ✨ など

## 🔧 プロンプトのカスタマイズ

### パラメータ一覧

| パラメータ | 説明 | デフォルト | 例 |
|-----------|------|----------|-----|
| {{title}} | 記事タイトル | - | "Reactの基本を学ぶ" |
| {{target_audience}} | 対象読者 | "一般" | "初心者" |
| {{purpose}} | 記事の目的 | "情報提供" | "学習" |
| {{keywords}} | キーワード | [] | ["React", "JavaScript"] |
| {{content_points}} | 内容要件 | - | "コンポーネントについて..." |
| {{word_count}} | 目標文字数 | 1500 | 2000 |

### プロンプトのバリエーション

**短い記事（1000文字）**
```
制約条件:
- 文字数: 1000文字程度
- 見出し数: 2〜3個
- 装飾: 最低1箇所
- 構成: 導入 → 本文 → まとめ（簡潔に）
```

**長い記事（3000文字）**
```
制約条件:
- 文字数: 3000文字程度
- 見出し数: 5〜7個
- 装飾: 5箇所以上
- 構成: 詳細に展開し、具体例を豊富に
```

## 🧪 プロンプトテスト結果

### テスト項目

1. **文字数精度**: 指定文字数の±10%以内
2. **装飾使用**: 指定数以上の装飾タグ使用
3. **見出し構造**: h2から開始、適切な階層
4. **読みやすさ**: 段落分け、リスト活用
